General

FAP_REQ_GEN_01 Creating object from specfication

Framework shall support creation object from specfication

Use cases

FAP_UC_001 Making changes in the object

There was a spec for object creation provided.
Object was created from the spec.
Some changes get required for the object.
There is mechanism of making change from dedicated change spec

FAP_UC_002 Creating env root from the spec

Preconditions

  • N/A

Steps

  • Clien specifies spec for FAP environment
  • Client creates environment with spec

Result

  • System creates environment automatically accordig with spec

Notes

  • In ver 0.0.1-3 the env is not created automatically but only on "manually" creating root from the correspondig node of spec

FAP_UC_003 Creating environment with dynamic number of objects

Preconditions

  • N/A

Action

  • Client creates environment where the number of objects can be dynamic. The objects withis the environment depends all other objects.
  • Client adds new object and connect all currently existing object wiht new one.

Reaction

  • System allows the client to create new connections without using static connections names.

Notes

  • This case is quite often in environments with many identical objects that connects all together.

States

FAP_REQ_STA_01 Transition by default

If no transion function nor operation is specified for the state then there shall the following default transition be done:

  • Copying the first input current value if it is compatible to state.

NOTE: Earlier one more default transition was used: copying current value of the state to new value. Denied currently.

FAP_REQ_STA_02 Formatting

State shall include APIs for formatting to string and from string

NOTES: To String formatting is required for logging. From string formatting is required for specifying initial values for the state (ref the section called “FAP_REQ_SPC_01 Initial value of state”)

Object specification

General

Refer to the section called “FAP_REQ_GEN_01 Creating object from specfication”

FAP_REQ_SPC_01 Initial value of state

There shall be possibility to specify initial value of state in object specification

FAP_REQ_SPC_02 State incapsulaton

Specification shall describe state with high level of incapsulation.

NOTE: this conforms with approach of representation the system as a set of state machines.

FAP_REQ_SPC_03 State named inputs

Specification shall describe state's inputs as named entities.

NOTE: This is required for simplifying implementation of transition function (it needs inputs identification) and connections between states.

Logging

FAP_REQ_LOG_01 Logging error on construction phase

Logging shall record errors happend on construction phase

FAP_REQ_LOG_02 Log file specifying

FAP API shall allow requesting the logging into specific log file.

If this is not requested then FAP shall use default log file.

FAP_REQ_LOG_03 Logging in transition functions

Framework shall allow using logging from transition function

FAP_REQ_LOG_04 Logging specification

Framework shall allow to specify logging parameters shall in two ways:

  • In object specification (ref the section called “Object specification”)
  • In dedicated logging specification

Embedded interpreter of transition function

FAP_UC_ET_01 Transition can use the accessible modules

The system defines transition for some state by specifying it within the state. This transition can use the module specifying within the system or super-system.

FAP_UC_ET_02 User can define functions

User defines function and then uses it within transition.

FAP_UC_ET_03 User can define custom data type

User defines custom data type, and operation for this data type

FAP_UC_ET_04 User can specify state of custom data type

User specifies the state of custom data type, ref the section called “FAP_UC_ET_03 User can define custom data type”

FAP_UC_ET_05 User can specify new type

User specifies the new type by specifying the constructor that results in required type.

Views

FAP_UC_VW_01 User specifies view for interpreter.

User runs DES interpreter and specifies how many views should be created and what kind they shoud be of.
